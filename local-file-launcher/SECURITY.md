# 安全性說明

## 網路存取限制

本機檔案啟動器具有多層安全保護機制，確保只有本機能存取服務。

### 1. 服務綁定限制 ✅

服務**只**綁定在 `127.0.0.1` (localhost/loopback)：

```python
app.run(
    host='127.0.0.1',  # 只監聽本機地址
    port=port,
    ...
)
```

**這表示：**
- ✅ 只有本機的程式可以存取 `http://127.0.0.1:8765`
- ❌ 區網內其他電腦**無法**存取（即使知道 IP 和 Port）
- ❌ 網際網路上的裝置**無法**存取
- ❌ 即使在同一台電腦，從外部 IP 也無法存取

### 2. 路徑安全驗證 ✅

所有檔案存取都經過 `safe_resolve` 函數驗證：

```python
def safe_resolve(base: str, rel: str) -> Optional[str]:
    # 防止路徑穿越攻擊
    # 確保只能存取設定的資料夾內的檔案
```

**防護內容：**
- 防止 `../` 路徑穿越
- 防止絕對路徑逃逸
- 防止符號連結攻擊

### 3. API 存取控制 ✅

- `/admin/usage` 報表頁面額外檢查來源 IP
- 只允許 `127.0.0.1` 和 `::1` (IPv6 localhost) 存取

### 4. 無 CORS 設定 ✅

已移除所有 CORS 相關設定：
- 不使用 `Access-Control-Allow-Origin: *`
- 服務只供同源存取（從 http://127.0.0.1:8765 本身）

## 安全性測試

### 測試 1：外部 IP 無法存取

```bash
# 在同一台電腦上，取得本機實際 IP
ipconfig

# 假設本機 IP 是 192.168.1.100
# 嘗試從實際 IP 存取（會失敗）
curl http://192.168.1.100:8765/
# 結果：Connection refused
```

### 測試 2：其他電腦無法存取

從區網內另一台電腦：
```bash
# 假設伺服器 IP 是 192.168.1.100
curl http://192.168.1.100:8765/
# 結果：Connection refused
```

### 測試 3：路徑穿越防護

```bash
# 嘗試存取上層目錄（會被擋）
curl -X POST http://127.0.0.1:8765/api/open \
  -H "Content-Type: application/json" \
  -d '{"rel": "../../../Windows/System32/config"}'
# 結果：{"ok": false, "error": "路徑不合法"}
```

## 最佳實踐建議

1. **使用 UNC 路徑**：存取網路資料夾時使用 `\\server\share` 格式
2. **定期更新**：使用 `scripts\update.bat` 取得安全更新
3. **最小權限**：執行服務的使用者只給予必要的檔案存取權限
4. **監控日誌**：定期檢查 `data/usage.sqlite3` 的存取紀錄
5. **防火牆**：確保 Windows 防火牆正常運作

## 風險評估

| 威脅類型 | 風險等級 | 防護措施 |
|---------|---------|---------|
| 外部網路存取 | ✅ 無風險 | 綁定 127.0.0.1 |
| 區網存取 | ✅ 無風險 | 綁定 127.0.0.1 |
| 路徑穿越 | ✅ 已防護 | safe_resolve 驗證 |
| CSRF 攻擊 | ✅ 低風險 | 僅本機存取 |
| XSS 攻擊 | ⚠️ 低風險 | 檔名需適當編碼 |

## 結論

本服務採用**零信任**網路模型，只允許本機存取，無法從外部網路或其他電腦存取檔案。CORS 設定已移除，確保最高安全性。
